What is a Broker?

  • A server in the Kafka cluster that stores and serves messages
  • Hosts one or more partition replicas of a topic
  • Each partition has exactly one leader broker at any time
  • Producers write to the leader; followers pull from it

What is Replication?

  • Each partition is copied across N brokers (replication-factor)
  • Here: replication-factor = 3 → P0 lives on Broker 0, 1, 2
  • Only the leader serves reads/writes; followers just sync
  • Guarantees no data loss if a broker dies

Leader Election on Failure

  • Controller detects broker heartbeat timeout
  • Picks a follower from the ISR as the new leader
  • Updates metadata — producers/consumers auto-redirect
  • When broker recovers, it rejoins as a follower and resyncs
